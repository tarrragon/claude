# 事件回應詳細規則

本文件包含事件回應的詳細分析步驟、多視角分析流程和報告格式。

> 精簡版（常駐載入）：.claude/rules/flows/incident-response.md

---

## 多視角並行分析

> 核心原則：複雜問題或原因不明的錯誤，可啟用多視角並行分析。
>
> 詳見：.claude/methodologies/multi-perspective-analysis-methodology.md

### 啟用條件

在 incident-responder 初步評估後，PM 可決定啟用：

| 條件 | 說明 |
|------|------|
| 原因不明確 | incident-responder 無法確定根因 |
| 認知負擔 > 10 | 問題複雜度高 |
| 風險等級 >= 高 | 高風險需要交叉驗證 |
| 跨模組影響 | 影響 3+ 模組 |
| 安全敏感 | 涉及認證、授權、敏感資料 |

### 多視角分析流程

```
incident-responder 初步評估
    |
    v
[PM 判斷] 需要多視角分析?
    |
    +-- 否（原因明確）--> 單一代理人 --> 派發修復
    |
    +-- 是 → 並行派發多個分析視角
              → 報告彙整（SA 整合）
              → PM 根據彙整報告派發修復
```

### 事件回應的標準視角組合

| 錯誤類型 | 必選視角 | 可選視角 |
|---------|---------|---------|
| 測試失敗（原因不明） | 程式碼、測試 | 架構、需求 |
| 編譯錯誤（複雜） | 程式碼、架構 | 環境 |
| 執行時錯誤（難重現） | 程式碼、環境 | 資料、架構 |
| 效能問題 | 效能、程式碼 | 架構 |
| 安全問題 | 安全、程式碼 | 架構 |

## 詳細執行步驟

### Step 1：收集錯誤上下文

- 錯誤訊息、發生時間、發生位置
- 相關檔案
- 重現步驟

### Step 2：分類錯誤類型

| 錯誤分類 | 子分類 | 識別特徵 |
|---------|-------|---------|
| 編譯錯誤 | 依賴問題 | "version solving", "package not found" |
| 編譯錯誤 | 類型錯誤 | "Type mismatch", "isn't a valid override" |
| 編譯錯誤 | 語法錯誤 | "Unexpected token", "Syntax error" |
| 測試失敗 | 測試本身問題 | 測試邏輯錯誤 |
| 測試失敗 | 實作與預期不符 | 實作行為錯誤 |
| 測試失敗 | 設計邏輯錯誤 | 需求理解錯誤 |
| 執行時錯誤 | 環境問題 | 環境變數、權限 |
| 執行時錯誤 | 資料問題 | 資料格式、遺失 |
| 執行時錯誤 | 程式錯誤 | 空指標、越界 |
| 效能問題 | - | 回應時間、資源使用 |

### Step 3：判斷根本原因

使用 5 Why 分析法追蹤根本原因。

### Step 4：安全敏感區域檢查

| 區域 | 識別關鍵字 | 風險等級 |
|------|-----------|---------|
| 認證 | login, auth, password, token, session | 高 |
| 授權 | permission, role, access, admin | 高 |
| 支付 | payment, transaction, billing | 嚴重 |
| 個資 | profile, personal, user data | 高 |

---

## 安全問題嚴重等級

| 等級 | 判定標準 | 處理時限 |
|------|---------|---------|
| **嚴重** | 繞過認證、資料外洩、遠端執行 | 24 小時內有計畫 |
| **高** | 權限提升、敏感資料存取 | 48 小時內修復 |
| **中** | 資訊洩漏、DoS 可能 | 本版本修復 |
| **低** | 最佳實踐問題、理論風險 | 排入後續版本 |

### Ticket 優先級對應

| 嚴重等級 | Ticket 優先級 | 特殊要求 |
|---------|--------------|---------|
| 嚴重 | P0 | 立即升級 PM、考慮暫停功能 |
| 高 | P1 | 優先排程 |
| 中 | P2 | 正常優先級 |
| 低 | P3 | 記錄技術債務 |

## Incident Report 格式

```markdown
# Incident Report

## 報告資訊
- **Incident ID**: INC-[YYYYMMDD]-[序號]
- **報告時間**: [時間]
- **嚴重程度**: 高 / 中 / 低

## 錯誤摘要
[一句話描述錯誤]

## 錯誤分類
- **主分類**: [分類]
- **子分類**: [子分類]

## 根本原因分析
[5 Why 分析]

## 安全風險評估
- **位於安全敏感區域**: 是/否
- **建議安全審查**: 是/否

## 派發建議
- **建議代理人**: [代理人名稱]
- **建議理由**: [理由]
```
