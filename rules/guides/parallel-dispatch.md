# 並行派發指南

本文件定義主線程進行並行任務派發的規則和條件。

---

## 核心原則

**無依賴任務可並行執行，提升開發效率。**

---

## 觸發條件

### 必須同時滿足的條件

| 條件 | 說明 | 檢查方法 |
|------|------|---------|
| 多任務 | 2+ 個待處理任務 | 同一 Wave 中有多個 pending Ticket |
| 無依賴 | 任務間無先後順序 | Ticket 間無 blockedBy 關係 |
| 無重疊 | 修改檔案無交集 | 檢查 Ticket 的 target 目錄 |
| 同階段 | 屬於同一 TDD 階段 | 都是 Phase 3b 或都是 Phase 1 |

---

## 並行安全檢查

啟動並行派發前，**必須**確認：

```markdown
## 並行安全檢查

- [ ] 檔案無重疊：各任務修改的檔案集合無交集
- [ ] 測試無衝突：各任務的測試可獨立執行
- [ ] 依賴無循環：任務之間無先後依賴關係
- [ ] 資源無競爭：不會同時存取相同外部資源
```

---

## 決策流程

```
任務分派
    |
    +-- 是單一任務?
    |   +-- 是 --> 單線程派發（標準流程）
    |   +-- 否 --> [並行判斷]
    |
    +-- [並行判斷]
        +-- 任務間有依賴?
        |   +-- 是 --> 依 Wave 序列派發
        |   +-- 否 --> [並行安全檢查]
        |
        +-- [並行安全檢查]
            +-- 檢查通過 --> 多線程並行派發
            +-- 檢查失敗 --> 降級為序列派發
```

---

## 並行派發格式

使用並行派發時，5W1H 格式：

```text
5W1H-{TOKEN}

Who: rosemary-project-manager (dispatcher)
What: 並行派發 N 個獨立任務
When: Wave X 開始
Where: {目標目錄1}, {目標目錄2}, ...
Why: 任務無依賴，可並行加速
How: [Task Type: Dispatch] 並行啟動 {代理人} x N

並行任務清單：
| Task | Ticket | Agent | Target |
|------|--------|-------|--------|
| 1 | {Ticket-ID-1} | {agent} | {target1} |
| 2 | {Ticket-ID-2} | {agent} | {target2} |

並行安全確認：
- [x] 檔案無重疊
- [x] 測試無衝突
- [x] 依賴無循環
- [x] 資源無競爭
```

## 數量原則

### 核心原則

**依任務性質判斷，無固定上限**

| 原則 | 說明 |
|------|------|
| 數量判斷 | 依任務性質決定，無固定上限 |
| 代理人類型 | 同一批次應為同類型代理人 |
| 失敗處理 | 單一失敗不影響其他任務繼續 |
| 結果彙整 | 全部完成後統一回報 |

### 大量並行適用情境

| 適用情境 | 說明 | 範例 |
|---------|------|------|
| 同一層級修改 | 修改都在同一架構層 | 所有 Repository 檔案 |
| 純重命名/格式化 | 無邏輯變更 | 變數重命名跨幾十個檔案 |
| 無邏輯依賴 | 各檔案獨立運作 | 獨立的 Widget 檔案 |
| 批量機械操作 | 規則明確的重複操作 | 導入路徑統一修正 |

---

## 不適用並行派發的情況

| 情況 | 原因 | 處理方式 |
|------|------|---------|
| TDD 跨階段 | Phase 順序必須遵守 | 序列派發 |
| 跨架構層修改 | 設計變更跨多層 | 序列派發 |
| 共享狀態 | 可能產生競爭條件 | 序列派發 |
| 有邏輯依賴 | 修改結果會影響其他任務 | 序列派發 |
| 依賴關係 | 前置任務必須先完成 | 依 Wave 序列 |
| 整合測試 | 需要完整環境 | 單獨執行 |

---

## 進度追蹤

並行派發後的追蹤機制：

| 追蹤項目 | 說明 |
|---------|------|
| 各任務獨立狀態 | pending / in_progress / completed / failed |
| 整體完成度 | X/N completed |
| 失敗處理 | 等待其他完成後統一處理 |

**進度顯示格式**：

```text
並行任務進度：[3/5 completed]
| Task | Ticket | Status | Agent |
|------|--------|--------|-------|
| 1 | 0.28.0-W1-001 | completed | parsley |
| 2 | 0.28.0-W1-002 | completed | parsley |
| 3 | 0.28.0-W1-003 | in_progress | parsley |
| 4 | 0.28.0-W1-004 | completed | parsley |
| 5 | 0.28.0-W1-005 | pending | - |
```
