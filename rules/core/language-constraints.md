# 語言約束規則

本文件定義專案中所有文字輸出的語言規範。

---

## 強制規則

### 規則 1：使用繁體中文

**所有回應必須使用繁體中文 (zh-TW)**

| 適用範圍 | 說明 |
|---------|------|
| 對話回應 | 所有與用戶的對話 |
| 文件撰寫 | 所有交接文件、工作日誌、Ticket |
| 程式碼註解 | Dart 檔案中的中文註解 |
| Commit 訊息 | Git commit 的描述部分 |

### 規則 2：禁用詞彙

以下詞彙**禁止使用**，必須替換為對應的正確用語：

| 禁用詞彙 | 正確用語 | 說明 |
|---------|---------|------|
| 智能 | Hook 系統、規則比對 | 避免過度擬人化 |
| 文檔 | 文件 | 台灣用語 |
| 數據 | 資料 | 台灣用語 |
| 默認 | 預設 | 台灣用語 |
| 代碼 | 程式碼 | 台灣用語 |
| 視頻 | 影片 | 台灣用語 |
| 軟件 | 軟體 | 台灣用語 |
| 硬件 | 硬體 | 台灣用語 |
| 信息 | 資訊 | 台灣用語 |
| 優化 | 最佳化 | 台灣用語（程式碼品質上下文可用「優化」） |

### 規則 3：技術術語保留原文

以下情況**保留英文原文**：

| 類型 | 範例 |
|------|------|
| 程式碼識別符 | `BookRepository`, `useCase`, `Widget` |
| 技術專有名詞 | Flutter, Dart, LSP, MCP, TDD |
| 指令和命令 | `/ticket create`, `git commit` |
| 檔案名稱 | `CLAUDE.md`, `pubspec.yaml` |
| API 和協定名稱 | REST API, WebSocket, JSON |

## 檢查機制

### 自動檢查

Hook 系統會在以下時機進行語言檢查：

| Hook | 檢查內容 |
|------|---------|
| PostToolUse (Write) | 檢查新建檔案是否包含禁用詞彙 |
| UserPromptSubmit | 提醒使用繁體中文回應 |

### 違規處理

| 違規類型 | 處理方式 |
|---------|---------|
| 使用禁用詞彙 | 提示修正，不阻止操作 |
| 使用簡體中文 | 提示修正，不阻止操作 |
