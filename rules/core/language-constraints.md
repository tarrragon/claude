# 語言約束規則

本文件定義專案中所有文字輸出的語言規範。

---

## 強制規則

### 規則 1：使用繁體中文

**所有回應必須使用繁體中文 (zh-TW)**

| 適用範圍 | 說明 |
|---------|------|
| 對話回應 | 所有與用戶的對話 |
| 文件撰寫 | 所有交接文件、工作日誌、Ticket |
| 程式碼註解 | 所有原始碼檔案中的中文註解 |
| Commit 訊息 | Git commit 的描述部分 |

### 規則 2：禁用詞彙

以下詞彙**禁止使用**，必須替換為對應的正確用語：

| 禁用詞彙 | 正確用語 | 說明 |
|---------|---------|------|
| 智能 | Hook 系統、規則比對 | 避免過度擬人化 |
| 文檔 | 文件 | 台灣用語 |
| 數據 | 資料 | 台灣用語 |
| 默認 | 預設 | 台灣用語 |
| 代碼 | 程式碼 | 台灣用語 |
| 視頻 | 影片 | 台灣用語 |
| 軟件 | 軟體 | 台灣用語 |
| 硬件 | 硬體 | 台灣用語 |
| 信息 | 資訊 | 台灣用語 |
| 優化 | 最佳化 | 台灣用語（程式碼品質上下文可用「優化」） |

### 規則 3：禁止使用 Emoji

**所有文字輸出禁止使用 emoji 符號**，適用範圍包括所有文件、對話和程式碼中的文字。

| 情境 | 禁止（用 emoji） | 替代方式 |
|------|----------------|---------|
| 標題/章節標記 | 標題開頭加裝飾 emoji（如目標符號 U+1F3AF） | `## Title` |
| 完成項目 | 列表項加完成 emoji（勾選符號 U+2705） | `- [x] done` |
| 未完成項目 | 列表項加未完成 emoji（叉號 U+274C） | `- [ ] pending` |
| 警告提示 | 文字前加警示 emoji（警示燈 U+1F6A8） | `[WARNING] 注意` |
| 狀態標示 | 狀態用色彩 emoji（紅綠圓 U+1F534/U+1F7E2） | `[高]` `[低]` |
| 程式碼正確示範 | 註解內加肯定 emoji（U+2705） | `// 正確：` |
| 程式碼錯誤示範 | 註解內加否定 emoji（U+274C） | `// 錯誤：` |

> **本表設計原則（W12-002.3）**：禁止欄不直接展示 emoji 字元，改用「裝飾類別 + Unicode codepoint」描述，避免本檔自動載入時污染 PM token attention pool（PC-072 → W12-002 → PC-080 鏈）。

**理由**：
- 確保跨平台渲染一致性
- 維持文件的專業性
- CLI 環境處理 emoji 可能產生對齊問題
- 避免不同系統顯示差異

### 規則 4：技術術語保留原文

以下情況**保留英文原文**：

| 類型 | 範例 |
|------|------|
| 程式碼識別符 | `BookRepository`, `useCase`, `Widget` |
| 技術專有名詞 | Flutter, Dart, LSP, MCP, TDD |
| 指令和命令 | `/ticket create`, `git commit` |
| 檔案名稱 | `CLAUDE.md`, `pubspec.yaml` |
| API 和協定名稱 | REST API, WebSocket, JSON |
