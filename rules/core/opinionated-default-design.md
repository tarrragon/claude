# Opinionated Default 設計原則（速查 stub）

> **完整規則**：`.claude/references/opinionated-default-design-details.md`（按需讀取，含案例、反模式對照表、判斷標準、與其他規則邊界）。本檔僅保留核心主張與觸發路由。

設計工具、規則、方法論、流程時，預設路徑必須引導正確做法。

## 四項核心主張

| 主張 | 核心 |
|------|------|
| 1 預設行為 > 文件規範 | 文件規範和工具預設不一致時，工具會贏。優先修工具 |
| 2 有預設值就表達 | 存在「多數情況下正確」的預設值時，工具必須表達建議式 opinion（提示 + 允許覆蓋） |
| 3 AI agent 場景加倍 | AI agent 沒有跨 session 記憶，工具即時引導是唯一可靠防線。opinion 從建議升級為必要 |
| 4 清理路徑主動掃描 | 前一次操作應同步但沒同步的狀態，工具必須主動掃描和報告 |

## 信號偵測

**每一個「寫文件提醒使用者遵守操作規範」都是信號** — 工具的預設行為還有改善空間。看到這個信號時，優先評估能否把提醒轉化為工具預設行為。

## 何時讀完整規則

| 情境 | 必讀 |
|------|------|
| 設計新 CLI / Skill / Hook / 流程 | 是（反模式對照表 + 設計檢查清單） |
| 審查既有工具的預設行為 | 是（案例 + 信號掃描步驟） |
| 撰寫規則/方法論發現需「寫文件提醒遵守」 | 是（信號偵測 + 改工具做法） |
| 日常開發、執行 Ticket | 否 |

## 檢查清單（精簡）

- [ ] 使用者需做選擇的地方，有「多數情況下正確」的預設值嗎？有 → 已表達建議？
- [ ] 現有文件中有「請遵守 X」「建議做 Y」的提醒？每個都是工具預設行為的改善信號
- [ ] 工具會被 AI agent 呼叫？上述每項優先級加倍
- [ ] 清理路徑：前次操作有應同步未同步的狀態嗎？
