# 實證決策軸發現方法論

本方法論定義「從累積錯誤記錄實證推導決策架構主導軸」的可重跑流程。決策架構不靠直覺 top-down 切層，而是先觀測系統實際卡在哪個失敗軸，圍繞主導軸設計防護層，並隨系統演進定期重新校準方向。

---

## 核心概念

決策架構的第一級軸**不可由直覺指定**。直覺傾向用功能/分類軸（這是什麼、路由到哪）top-down 切層，但真實的決策失敗常集中在「信念是否對齊世界」這類非功能軸上。本方法論把軸的判定交給累積錯誤語料：壓力測或盤點摩擦 → 挖 error-pattern 語料 → 七軸分類 → 實證判定主導軸 → 圍繞主導軸分流設計防護 → bottom-up 登錄。

**Why**：直覺切層會把資源投在沒卡過的軸（如功能分類），而真實漏口（如知識缺口、狀態漂移）持續失血。資料指向哪裡，防護就該裝哪裡。

**Consequence（不這樣做的後果）**：圍繞錯誤軸建護欄會產生「防護完整卻持續同類失敗」的假象——框架在 F 軸裝最多協議卻仍是最大軸，正因為若不先實證就無法看出協議型防護的漏口在哪。

**Action**：任何「決策樹體系化 / 決策路由設計 / 防護層規劃」需求，先跑本方法論的六步得出當前軸快照，再設計層，而非直接憑經驗切層。

---

## 七軸分類框架

語料每條歸一主軸（跨軸條目取最直接根因為主軸，次軸另註）。

| 軸 | 名稱 | 判準 |
|----|------|------|
| A | 功能/分類 | 這是什麼輸入、路由到哪個 handler；分類錯/路由錯 |
| B | 權威/確認 | 誰有權決定、何時該確認用戶、權限邊界 |
| C | 可逆性/破壞 | 不可逆/破壞性動作、是否先確認/先備份 |
| D | 狀態完整性（記錄 vs 世界） | confabulation、幻覺結果、stale 狀態、未提交、漂移、假完成、記錄≠世界 |
| E | 並行/worktree | worktree 隔離、平行 session、檔案衝突、race |
| F | 知識/規格缺口 | 缺 context、規格漏、需求模糊、重複造輪、未查既有就斷言 |
| G | 實作/品質（非決策軸，兜底） | 純程式品質、格式、命名等與決策路由無關者 |

> A/G 屬「實作/品質類」軸，與「決策該卡哪軸」關聯較弱，判定主導軸時可作扣除子集對照（見步驟 4）。

## 執行步驟（六步，可重跑）

### 步驟 1：觀察摩擦或取既有語料

壓力測子系統、盤點開發摩擦，或直接取既有 error-pattern 全集為語料。**樣本來源決定結論可信度**：單一 session / 單一場景的樣本有並行偏誤（見步驟 4），跨專案累積語料才接近不偏。

### 步驟 2：挖 error-pattern 語料

對 `.claude/error-patterns/` 全類目錄取全部條目。檔名 slug 為主要分類依據；語意模糊者讀檔頭 title / 根因判定。記錄語料快照日期與總筆數，供他人依同條目集重跑核對。

### 步驟 3：七軸分類

每條 PC/IMP/ARCH/DOC/... 歸一主軸，產出各軸計數 + 佔比。邊界跨軸條目依「最直接根因」歸主軸並標註顯著次軸。分類為判斷而非機械事實，須聲明預期重跑差異區間（如邊界 15-20 條 ±1）。

### 步驟 4：主導軸判定 + 樣本偏誤查驗（WRAP Step0）

判定主導軸前，先過 WRAP Step0 資料充足度閘門：

- **樣本量充足？** 單場 N 小（如 10）的結論須被大樣本（如 326）驗證或修正後才採信。
- **樣本是否偏誤？** 並行 session / 同質任務會放大單場高頻軸、壓抑跨任務分散的長尾高總量軸。判定時對照「全語料分佈」與「扣除 A/G 的決策相關子集分佈」兩個視角。
- **Consider-the-Opposite**：主動找與當前主導軸相反的證據（「如果主導軸不是 X 會怎樣」），避免確認偏誤錨定。

### 步驟 5：分流設計（手段隨軸不對稱）

主導軸確定後，依軸的可機械化程度分流設計防護手段。**體系化不等於對所有軸用同一手段**。

| 軸性質 | 防護手段 | 範例 |
|--------|---------|------|
| 難機械化軸（要求行動者察覺「我在斷言沒查過的事」） | 判斷協議（勸告型，行動前須 discharge 的具名狀態） | F 軸：行動前最小實證（--help、實機跑一次、ls 二元查證、掃既有） |
| 可驗證軸（believed-state vs world 可用固定值比對） | 機械閘門（強制型 hook） | D 軸：`git show ref:` 固定值、commit/complete 一致性 hook |

> **手段不對稱是核心**：難機械化軸即使裝最多防護仍可能是最大漏口，因協議是勸告型；不可誤以為「裝了協議＝堵住了」。

### 步驟 6：bottom-up 軸登錄

新防護閘門誕生時，宣告其所屬軸並登錄組織，而非 top-down 預先切層。組織方式是「子系統壓力 → 露出某軸 → 守該軸 → 登錄」。新軸由真實摩擦露出，不由設計者憑空指定層級。

---

## 重新校準 trigger（綁定，非 trigger-less）

軸分佈是**用本方法跑一次得到的當前快照**，model 演進與語料增長會改變分佈。校準必須綁定明確 trigger，禁止「以後再校準」式的無 trigger 延後（違反 decision-trigger-binding 規則）。

| Trigger | 校準動作 |
|---------|---------|
| model 大版本升級 | 重跑步驟 1-4，比對軸分佈是否位移 |
| 語料新增 N 條（建議 N=50 或語料量 +15%） | 重跑步驟 2-4 計數 |
| 固定週期（如每季 / 每大版本） | 全六步重跑 |
| 新子系統上線並經壓測 | 步驟 1 補語料 + 步驟 6 登錄新露出的軸 |

> 量化閾值（N 條 / 週期）本身不是合法 trigger，須先包裝為監測 ticket 後生效（decision-trigger-binding 規則 2）。

---

## 反 anchor 原則

- **輸出軸是快照，永不寫死**：任何「主導軸是 X」的結論都帶快照日期與樣本量上下文，文件不得將某軸固化為永久前提。
- **Consider-the-Opposite**：每次判定主導軸時主動尋找反面證據，防確認偏誤。
- **樣本偏誤防護**：單場 / 並行樣本結論一律標記「待大樣本驗證」，不得直接作為架構決策依據。

---

## 檢查清單

- [ ] 語料來源已記錄（快照日期 + 總筆數），可由他人重跑核對
- [ ] 七軸分類完成，含各軸計數 + 佔比 + 預期重跑差異區間
- [ ] 主導軸判定已過 WRAP Step0（樣本量充足 + 偏誤查驗 + Consider-the-Opposite）
- [ ] 分流設計已區分難機械化軸（判斷協議）與可驗證軸（機械閘門）
- [ ] 新閘門已 bottom-up 登錄所屬軸
- [ ] 重新校準 trigger 已綁定（model / 語料量 / 週期 / 新子系統），非 trigger-less
- [ ] 輸出軸標記為快照，未被固化為永久前提

## 附錄：1.2.0 實例

本方法論的耐久資產是「方法本身」；下列 F+D 結論是 1.2.0（語料快照 2026-06-18，N=326）用本方法跑一次的當前快照，非永久結論。

**全語料軸分佈（N=326，跨專案不偏）**：

| 軸 | 計數 | 佔比 |
|----|------|------|
| F 知識/規格缺口 | 97 | 29.8% |
| D 狀態完整性 | 78 | 23.9% |
| A 功能/分類 | 62 | 19.0% |
| G 其他/品質 | 47 | 14.4% |
| B 權威/確認 | 22 | 6.7% |
| C 可逆/破壞 | 11 | 3.4% |
| E 並行/worktree | 9 | 2.8% |

**判定結果**：F（29.8%）為最大軸、D（23.9%）緊隨為第二，雙主導共佔 53.7%。F 軸內部以 F4「未驗假設就行動」（39.2%）為首；F4 與 D 是「信念 vs 世界」斷層的事前/事後兩面。

> **可重跑邊界**：本方法論的七軸框架（步驟 3）只定義 7 個**主軸**，七軸計數（如 F=29.8%）可依本文件機械重跑。F 軸**子軸**（F1–F6，含 F4=39.2%）屬「對主導軸再深挖」的選用追加分析，其拆分規則見 1.2.0-W1-032 附屬 F-drill 子分析，不在主框架——故重跑本方法論會穩定得出「F 為最大軸」，但子軸佔比需另跑一次同規格子分析才能重現。

**樣本偏誤被大樣本修正的實例**：1.2.0 單一並行 session 樣本（N≈10）原判「D 唯一主導」。經 326 大樣本對照，D 被**驗證**為真實高頻軸（非並行偶然），但「D 唯一主導」被**修正**——F 才是最大軸。並行 session 放大單場高頻軸（D）、壓抑跨任務分散的長尾高總量軸（F），正是步驟 4 WRAP Step0 樣本偏誤的具體形態（326 vs 10 對照）。

**分流設計結果**：F 軸（難機械化）→ 判斷協議（WRAP Step0 + saffron 防重複 + tool-output-trust 規則 3）；D 軸（可驗證）→ 機械閘門（028/030 hook + tool-output-trust 規則 5）。E 軸全語料僅 2.8%，證實其為真實但小眾的軸，不應升為第一級。
