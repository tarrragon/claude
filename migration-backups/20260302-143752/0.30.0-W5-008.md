# 0.30.0-W5-008: 檢視 agent: coriander-integration-tester

## 目標
依照 skill-design-guide 新規範重新檢視 agent `coriander-integration-tester`，評估是否需要重寫或調整。

## 檢視對象
- 檔案: `.claude/agents/coriander-integration-tester.md`
- 類型: Agent 定義

## 評估結果

**合規狀態**：需要重寫

### 發現項目

| # | 檢查項目 | 狀態 | 說明 |
|---|---------|------|------|
| 1 | YAML frontmatter 必填欄位 | 合規 | `name` 和 `description` 皆存在 |
| 2 | YAML 自訂屬性 | 違規 | `tools` 和 `color` 非標準欄位，應移除或改用 `allowed-tools` |
| 3 | description 結構 | 部分合規 | 包含「做什麼」和「何時使用」，但使用英文撰寫，與專案繁體中文規範不一致 |
| 4 | 大小限制（500 行） | 違規 | 共 550 行，超過 500 行上限 |
| 5 | Progressive Disclosure | 違規 | 大量詳細內容（報告模板、檢查清單、協作細節）未拆分到 references/ |
| 6 | 禁止 emoji | 違規 | 第 73、75 行使用 emoji 符號（書本圖示） |
| 7 | 禁止中英文重複 | 違規 | 多處中英文重複說明同一件事，如「核心職責」段落與「Core Integration Testing Principles」段落、「整合測試執行準則」與「When designing integration tests」段落內容大量重疊 |
| 8 | YAML 多行語法 | 合規 | 未使用 `\|` 或 `>` 語法 |
| 9 | 搜尋工具區段 | 建議移除 | 尾部的「搜尋工具」區段應由 AGENT_PRELOAD 統一處理，不應在每個 agent 中重複 |
| 10 | 語言一致性 | 違規 | 大量段落使用英文撰寫（第 79-111、213-260、300-320 行），應統一為繁體中文 |

### 結論

**需要重寫**。此 agent 定義有以下結構性問題：(1) 超過 500 行大小限制；(2) 中英文內容大量重複，同一職責用中文和英文各說一次；(3) 含有 emoji；(4) YAML frontmatter 使用非標準欄位。需要精簡合併重複內容、將詳細模板和檢查清單拆到 references/、統一語言為繁體中文。

### 具體調整項目

1. **YAML frontmatter**：`tools` 改為 `allowed-tools`，移除 `color`；`description` 改為繁體中文
2. **合併中英文重複**：將「Core Integration Testing Principles」合併到「核心職責」中文段落，移除英文重複
3. **移除 emoji**：第 73、75 行的書本圖示改為純文字
4. **拆分到 references/**：「整合測試報告模板」「Integration Testing Checklist」「Success Metrics」移到 `.claude/agents/references/coriander-integration-tester/`
5. **移除搜尋工具區段**：由 AGENT_PRELOAD 統一處理
6. **移除「敏捷工作升級機制」英文段落標題**：統一為繁體中文
7. **精簡目標**：550 行精簡至 250-300 行以內
