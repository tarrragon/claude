# 0.30.0-W5-016: 檢視 agent: oregano-data-miner

## 目標
依照 skill-design-guide 新規範重新檢視 agent `oregano-data-miner`，評估是否需要重寫或調整。

## 檢視對象
- 檔案: `.claude/agents/oregano-data-miner.md`
- 類型: Agent 定義

## 評估結果

**合規狀態**: 需要調整

### 發現項目

| # | 檢查項目 | 狀態 | 說明 |
|---|---------|------|------|
| 1 | frontmatter 標準欄位 | 不合規 | `tools`, `color`, `model` 為非標準欄位，可能導致解析錯誤。標準欄位僅有 `name`, `description`, `license`, `compatibility`, `allowed-tools`, `metadata` |
| 2 | description 第三人稱 | 合規 | 使用第三人稱描述 |
| 3 | description 結構 | 部分合規 | 包含「做什麼」但缺少明確「何時使用」觸發條件關鍵字 |
| 4 | description 長度 | 合規 | 未超過 1024 字元 |
| 5 | 禁止 XML 角括號 | 合規 | frontmatter 無 `<` `>` |
| 6 | 禁止 YAML 多行語法 | 合規 | 無 `\|` 或 `>` block scalar |
| 7 | 禁止自訂屬性 | 不合規 | `tools`, `color`, `model` 為自訂屬性 |
| 8 | 檔案大小 | 不合規 | 681 行，超過建議的 500 行上限 |
| 9 | Progressive Disclosure | 不合規 | 大量內容（輸出格式範本、協作細節、搜尋工具區段）應移至 references/ |
| 10 | emoji 使用 | 不合規 | 內文使用 emoji（如行首 emoji 標題） |
| 11 | 內容重複 | 不合規 | 英文和中文內容重複描述相同職責（核心職責中文版 + Core Data Extraction Principles 英文版） |
| 12 | 搜尋工具區段 | 不合規 | 附加的「搜尋工具 ripgrep」區段應為共用 references/ 或 AGENT_PRELOAD 內容，不應每個 agent 重複 |

### 結論

需要重寫。主要問題：(1) frontmatter 使用非標準欄位 (2) 檔案過長且中英文內容重複 (3) 缺少 Progressive Disclosure 架構。

### 具體調整項目

1. **frontmatter 修正**：移除 `tools`, `color`, `model`，改用 `allowed-tools` 和 `metadata` 標準格式
2. **description 改善**：加入明確觸發關鍵字（如 "Use when user mentions 'web scraping', 'data extraction', 'external research'"）
3. **內容精簡**：合併中英文重複內容，僅保留中文版本
4. **Progressive Disclosure**：將輸出格式範本、協作細節移至 `references/`
5. **移除搜尋工具區段**：此區段應整合到 AGENT_PRELOAD 或共用 reference
6. **移除 emoji**：符合文件格式規則
