### 合規狀態: 需調整

### 發現項目
| # | 檢查項目 | 狀態 | 說明 |
|---|---------|------|------|
| 1 | YAML frontmatter | OK | 有 `---` 分隔符，`name` 和 `description` 皆存在 |
| 2 | description 結構 | 需調整 | 缺少「何時使用」觸發條件。目前僅說明功能（"Automatically extract reusable patterns..."），未指定觸發場景或關鍵字 |

### 結論
主要問題是 description 缺乏觸發條件。由於此 Skill 主要作為 Stop hook 自動執行，應在 description 中明確說明觸發方式和使用場景。另外，description 應改用雙引號包裹以遵循最佳實踐。

### 具體調整項目（如需要）
1. 修改 description 加入觸發條件，改用雙引號包裹：
   - 建議：`description: "Automatically extracts reusable patterns from Claude Code sessions and saves them as learned skills. Runs as a Stop hook at session end. Use when: (1) configuring session-end learning, (2) reviewing extracted patterns, (3) managing learned skills in .claude/skills/learned/."`
2. 考慮設定 `user-invocable: false` 因為此 Skill 主要由 Hook 觸發而非用戶手動呼叫
