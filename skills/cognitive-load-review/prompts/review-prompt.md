## 常見問題快速診斷

### 問題: 函式過長

```
診斷: 函式超過 20 行
原因分析:
- 混合多個責任?
- 重複邏輯?
- 過度細節?

建議方案:
1. 識別獨立區塊
2. 提取為子函式
3. 命名說明意圖
```

### 問題: 深層巢狀

```
診斷: 巢狀超過 3 層
原因分析:
- 多層 if-else?
- 迴圈內條件?
- 例外處理?

建議方案:
1. 早期返回 (Guard Clause)
2. 提取內層函式
3. 策略模式
```

### 問題: 模糊命名

```
診斷: 使用 data/info/value 等詞
原因分析:
- 趕時間?
- 不確定用途?
- 缺乏領域知識?

建議方案:
1. 問「這是什麼？」
2. 用具體名詞
3. 加入領域詞彙
```

## 評分標準

### 維度評分說明

| 分數 | 含義 | 標準 |
|------|------|------|
| 5 | 優秀 | 完全符合最佳實踐 |
| 4 | 良好 | 小問題，不影響理解 |
| 3 | 可接受 | 有改善空間，但可用 |
| 2 | 需改進 | 明顯問題，建議修改 |
| 1 | 不可接受 | 嚴重問題，必須修改 |

### 總評對照

| 總分 | 等級 | 建議 |
|------|------|------|
| 21-25 | 優良 | 可直接合併 |
| 16-20 | 良好 | 可合併，記錄改善項 |
| 11-15 | 可接受 | 建議修改後合併 |
| 6-10 | 需改進 | 必須修改後重新審查 |
| 1-5 | 不可接受 | 重寫 |
